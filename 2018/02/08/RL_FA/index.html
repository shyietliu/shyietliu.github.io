5. Function Approximation

In previous posts, we introduced 3 methods to achieve prediction and control — Dynamic Programming (DP), Monte-Carlo (MC) method, Temporal Difference (TD) method. All of them are tabular, which means that all the information we need (e.g. \(V(s_t)\) for a especific state) is stored in a table. Once we need one, we look it up in that table.

However, in practical, it's hard to build a table for large value space. To overcome this shortcoming, we utilize function approximation (FA).

The basic idea of FA is approximating the information (says, value function) in a supervised way. In other words, FA use certain supervised approach to map state feature to value function. For instance, one could use a neural network to do that.

5.1 Linear Method

One of the most important special cases of gradient-descent function approximation is that in which the approximate function, \(V_t(s)\), is a linear function of the parameter, \(X(s)\). \[ V(s)=\boldsymbol{w}\boldsymbol{x}(s) \] Where \(\boldsymbol{x}(s)\) is a vector, denotes feature for a state and it also called parameter. \(\boldsymbol{w}\) is a learned parameter called weight. We use Gradient Descent method to update the the value function (essentially, the weight).

One important problem here is feature extraction, i.e., what feature we are going to use.

There are some classical methods: Coarse Coding, Tile Coding, Radial basis functions (RBFs) and Kanerva Coding. Details can be found here.

5.2 Control with FA

\[ \begin{equation} \begin{split} \boldsymbol{w}&=\boldsymbol{w}+\alpha[V_t-Q_t(s_t,a_t)] \nabla_{\boldsymbol{w_t}}Q_t(s_t,a_t)\\ &=\boldsymbol{w}+\alpha\delta_t\boldsymbol{e_t} \end{split} \end{equation} \]

Where \(\delta_t=R_{t+1}+\gamma Q_t(s_{t+1},a_{t+1})-Q_t(s_t,a_T)\), \(\boldsymbol{e_t} =\gamma \lambda \boldsymbol{e_{t-1}}+\nabla_{w_t}Q_t(s_t,a_t)\) and \(\boldsymbol{e_0}=0\).
